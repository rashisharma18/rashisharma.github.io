Penetration Testing: Web Application

About the Project

In this project, a vulnerable web application hosted on a Linux server was provided, with the goal of conducting an extensive Penetration Test to recover 6 secret files from the system, and discover security vulnerabilites present on the system. The final results were presented in a thorough Technical Report, outlined with an Executive Summary, including specific security recommendations with a detailed plan of action to mitigate the vulnerabilities present on the system.

Solution and Approach

The vulnerable Web Application, deployed on a Linux Server was provides as a Virtual Machine image. A Kali machine was used as the attacking machine.

Approach

Network Enumeration: First, a complete network and port analysis was conducted to map IP addresses, and disocver the services running on each machine.
Webpage Reconnasaince: The Web Application hosted on the Linux Server was analyzed to disocver web page vulnerabilties.
Initial Access: Inital access was gained into the system by exploiting an existing File Upload vulnerability using a custom reverse-shell script.
Flags: The "Secret Files" were obtained by using the following methods:

  • Brute Force Password Cracking
  • SQL Injection
  • Phishing Campaign launched against the creator of the Web Page
  • Reverse software engineering of an obfuscated executable
  • Thorough Web Page Analysis

Skills

Here are the skills I used and learned during this project

  • Brute Forcing
  • Phishing
  • File Upload Vulnerability
  • SQL Injection
  • Reverse Engineering
  • OWASP Top 10
  • Tools Used: GoPhish, Metasploit, SQLMAP

Impact

This project helped me get hands-on experience in conducting a comprhensive Web Application Penetration Testing exercise. This helped me learn about the OWASP Top 10 vulnerabilities, and hands on experience conducting Phishing exercises.
