Offensive and Defensive Analysis of Lockheed Martin Kill Chain

About the Project

In this project, a vulnerable application was deployed on Amazon Web Services and the seven steps of the Lockheed Martin Kill Chain were executed, both from the attacker's and defender's perspectives. A threat model of the deployed application was created using the STRIDE method. Finally, an analysis of the results was drawn and countermeasures were proposed to secure the application. The final results also validated the accuracy of the threat model, and tabulated if the vulnerabilities exploited in the Offense stage were reported or not.

Solution and Approach

Vulnerable Application Deployment: A web application vulnerable to Brute Force, SQL Injection and Command Injection was deployed on the AWS Cloud.
Thrat Analysis: A complete Threat Analysis on the Web Architecture was conduced using the Microsoft Threat Modelling Tool, mapping threats into the STRIDE Framework.

  • User View: The deployed application was analyzed from the user view, and a Threat Model was developed assuming a threat originating from outside the environment.
  • Administration View: The deployed application was analyzed from the administration view, and a Threat Model was developed assuming a threat originating from within the production environment.
  • Custom Threat Modelling Template: The Microsoft Threat Modelling Tool does not contain a stencil for Amazon Web Services resources. Therefore, a new stencil was created to include AWS resources. This involved adding the resources, mapping the threats associated with each resource (S3, EC2), and categorizing the threats into the existing STRIDE Framework. The resulting stencil was then merged with the existing stencil to create a comprehensive Threat Model to accurately map the threats associated with the deployed architecture.
Offensive Analysis: An attack was consucted on the deployed Web Application, following the steps of the Lockheed Martin Kill Chain. This analysis provided an opportunity to map an attacker's TTPs with the Cyber Kill Chain.
Defensive Analysis: A step-by-step Defensive Analysis was conducted to propose security recommendations to prevent exploitation at each stage fo the Kill Chain.
